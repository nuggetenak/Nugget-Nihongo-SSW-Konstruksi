Validasi yang dicakup:

### B. Performa bundling: **IMPROVED (material)**
Perubahan bundling menggunakan `manualChunks` memisahkan:
- `vendor` chunk
- `cards-data` chunk (data statis besar)
- chunk mode per fitur (hasil lazy-loading)

Dampak utama:
- JS main entry menurun signifikan dibanding kondisi pra-audit.
- Data kartu besar kini berada pada chunk terpisah sehingga tidak menumpuk di satu file utama.

### C. Security dependency audit: **BLOCKED BY ENV**
`npm audit --omit=dev` gagal bukan karena finding package, namun karena endpoint npm advisory mengembalikan **403 Forbidden** di environment ini.

## 4) Remediasi yang Diterapkan di Pass 2
1. Tambah script audit integritas (`scripts/audit-integrity.mjs`).
2. Tambah script `audit:integrity` + `audit:full` di `package.json`.
3. Aktifkan `manualChunks` di `vite.config.js` untuk split `vendor` dan `cards-data`.

## 5) Risiko Residual
- Warning chunk size masih muncul (threshold default 500 kB), terutama pada data cards sebagai dataset utama.
- Ini bersifat expected untuk aplikasi berbasis data besar offline-first.
